# 実際に使うときの流れ

## 運用フロー

1. **社内側のアカウントをつくる**（管理者・設計・IC）
2. **設計もしくはICがお客様のアカウントをつくる**
3. **設計もしくはICがお客様情報を入力する**
4. **設計もしくはICがお客様にURL発行を行う**（この際、ログイン方法なども伝達する）
5. **お客様がこのシステムを自宅で使い、仕様をある程度決めてもらう**
6. **打合せ前にICがお客様の利用状況を確認する**
7. **ICとお客様との打合せでこのシステムを見ながら、打合せし、入力をする**
8. **全てが確定すれば、仕様決定ボタンを押し、見積書と仕様書を出力する**

---

## システム要件

### 1. データ管理
- 全アイテムはSupabaseに繋がっていること（Localで管理しない）
- 全アイテムはカテゴリの中で管理されていること

### 2. カテゴリ階層構造
| 階層 | 説明 | 例 |
|------|------|-----|
| 大カテゴリ | タブレベル | 設計・外装・内装・水廻り設備 |
| 中カテゴリ | サイドバーレベル | 外壁・屋根・玄関ドア |
| 小カテゴリ | フィルターレベル | 窯業系サイディング・ガルバリウム鋼板・塗り壁 |

### 3. 画面構成
- **ヘッダー**: 上部のナビゲーション
- **サイドバー**: 左側のカテゴリ一覧
- **メイン**: 中央の商品表示エリア

### 4. 価格設定
- 商品によって価格が違う物もあれば、全商品同じものもある
- **標準**: 0円で選択できるもの
- **オプション**: 追加で1円でもかかるもの

### 5. 単位
- アイテムの単位はバラバラ
- 全て日本語で表記: 個・セット・m・㎡・一式・ヶ所など
- 英語を使わない
